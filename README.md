# Lonely Planet: Star Citizen

> Unofficial travel guides to the 'Verse.

Curated attractions, hidden gems, and essential travel tips for the systems you fly through. Built in the editorial spirit of classic travel writing — for explorers, by explorers.

---

## Quick Start

```bash
# Install dependencies
npm install

# Start the development server (with hot reload)
# → Opens at http://localhost:4321
npm run dev

# Build for production
npm run build

# Preview the production build
# NOTE: Must run `npm run build` first!
npm run preview
```

---

## Available Scripts

| Command | Description |
|---|---|
| `npm run dev` | Starts the Astro dev server at `http://localhost:4321` (hot reload enabled). |
| `npm run build` | Builds a static version of the site into the `dist/` folder. |
| `npm run preview` | Serves the built `dist/` folder locally for testing. Requires `build` first. |
| `npm run astro` | Direct access to the Astro CLI. |

## Project Structure

```
├── src/
│   ├── components/     # Reusable UI components (AttractionEntry, ExplorerView, etc.)
│   ├── layouts/        # Page templates (PlanetLayout, POILayout)
│   ├── pages/          # Content pages (Astro routes)
│   └── styles/         # Global CSS and design tokens
├── public/
│   └── images/         # Static image assets
├── docs/               # Reference docs & content planning
├── dist/               # Build output (generated by `npm run build`)
└── package.json
```

---

## Why no `npm start`?

This project uses Astro's modern scripts (`dev`, `build`, `preview`). There is no `start` script defined — use **`npm run dev`** for development or **`npm run preview`** to check a production build.

## Tech Stack

- **Framework**: [Astro](https://astro.build/)

- **Content**: Markdown / MDX
