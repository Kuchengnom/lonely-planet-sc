     1|# Visual Language: Lonely Planet Star Citizen
     2|
     3|## 📖 Design DNA (Print Inspiration)
     4|The goal is to replicate the "dense but organized" feel of a Lonely Planet print guide.
     5|
     6|### 1. Layout & Grid
     7|- **Multi-Column Asymmetry**: Move away from centered web layouts. Use a mix of wide main content and narrow sidebars (call-outs).
     8|- **The "Fast-Fact" Sidebar**: Every POI page should have a sidebar containing:
     9|    - **Coordinates/Location**: (e.g., MicroTech -> North Pole)
    10|    - **Difficulty**: (Easy/Tourist vs. Hard/Explorer)
    11|    - **Required Gear**: (e.g., Cold weather suit, specific ship size)
    12|    - **Vibe**: (e.g., "Eerie", "Majestic", "Hidden")
    13|- **Visual Anchors**: Large, bleeding-edge images paired with small, captioned "detail" photos.
    14|
    15|### 2. Typography
    16|- **Headings**: Bold, sans-serif, high-impact. Using a "Travel Guide" style font that feels authoritative yet adventurous.
    17|- **Body**: Clean, readable serif or humanist sans-serif for long-form storytelling.
    18|- **Captions**: Small, italicized, tight leading.
    19|
    20|### 3. Iconography & Signposting
    21|- **The "Star-Rating"**: Use a custom icon (maybe a star-ship or a planet) to rate POIs.
    22|- **Category Icons**: 
    23|    - 🎄 `Easter Egg`
    24|    - 🏔️ `Natural Wonder`
    25|    - 🏢 `Urban Sight`
    26|    - 🛸 `Technical Curiosity`
    27|- **Color Coding**: Each star system gets a distinct "Edition" color (e.g., Stanton = Blue/White, Pyro = Orange/Red).
    28|
    29|## 🛠️ Digital Translation
    30|- **CSS Grid/Flexbox**: Use CSS Grid to create the magazine-style columns.
    31|- **Responsive Strategy**: On mobile, the sidebars stack, but on desktop, they maintain the asymmetric "book" look.
